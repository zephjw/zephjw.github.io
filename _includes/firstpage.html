The Weight Tracker

Android Health Application – Kotlin | MVVM | Room | Jetpack

Software Design and Engineering

The Weight Tracker is an Android application built in Kotlin during my third year of the Computer Science program. The application allows users to log body weight, visualize trends using a line chart, and manage health-related records such as glucose, insulin, and sleep data.

The application follows the MVVM architecture pattern using:

  • Room for local persistence
  • Fragments for UI composition
  • ViewModel for state management
  • Repository pattern for separation of concerns

Key Enhancements

  • Replaced hardcoded dp dimensions with responsive constraints.
  • Replaced ListView with MaterialButtonToggleGroup for proper Material Design compliance.
  • Fixed missing android:id attributes causing potential NullPointerExceptions.
  • Implemented a proper Repository layer between ViewModel and DAO.
  • Fixed critical bug where android.R.attr.height was incorrectly stored in SharedPreferences.

Reflection

This enhancement process reinforced the distinction between code that runs and code That is architecturally correct. Many issues were not immediately visible but would surface under device variation or scaling conditions.

Database Engineering (Room + SQLite)

The database layer required a significant redesign. The original DAO contained invalid Room annotations, improper return types, duplicate signatures, and incorrect parameter bindings.

Database Enhancements

  • Redesigned InitialUserData entity using Float and a type-safe Gender enum.
  • Added foreign key relationship with CASCADE delete rule.
  • Added index on userId for query performance.
  • Created TypeConverter for enum persistence.
  • Reduced DAO from 30+ incorrect methods to minimal required operations.
  • Converted DAO methods to suspend functions.
  • Implemented reactive queries using Flow<List<InitialUserData>>.

Algorithms and Data Structures

Search, Filter, and Sort


    fun searchByDate(query: String) {
        if (query.isBlank()) {
            loadAllEntries()
            return
        }
    
        viewModelScope.launch {
            val results = repository.searchByDate(query)
            _entries.value = applySortOrder(results, _sortOrder.value)
            _isFiltered.value = true
        }
    }

Supports partial matches:

  • "2024" → Returns all entries from that year
  • "2024-03" → Returns entries from March 2024

Search Strategy: SQL LIKE pattern matching

Time Complexity: O(n) worst case

In-Memory Sorting (Timsort)


    fun toggleSortOrder() {
        val newOrder = if (_sortOrder.value == SortOrder.ASCENDING)
            SortOrder.DESCENDING else SortOrder.ASCENDING
    
        _sortOrder.value = newOrder
        _entries.value = applySortOrder(_entries.value, newOrder)
    }

Algorithm: Kotlin Timsort

Time Complexity: O(n log n) worst case

Best Case: O(n) (nearly sorted data)

Linear Search (Exact Date Match)


    fun findFirstEntryByExactDate(date: String): WeightEntry? {
        for (entry in _entries.value) {
            if (entry.date == date) return entry
        }
        return null
    }

Best Case: O(1)
Worst Case: O(n)

For larger datasets, a HashMap-based index could reduce lookup to O(1).
